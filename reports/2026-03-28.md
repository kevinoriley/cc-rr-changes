## red-run Impact Analysis

### High Impact (requires action)

- **[v2.1.85] PreToolUse hooks can satisfy AskUserQuestion via `updatedInput`** —
  Impact: This enables headless/automated approval of AskUserQuestion prompts.
  red-run's approval gates use AskUserQuestion extensively. A PreToolUse hook
  could auto-approve routing decisions, enabling fully autonomous operation
  without operator interaction.
  Action: Consider adding an optional hook-based auto-approve mode for CTF
  engagements where the operator wants zero-touch execution.

- **[v2.1.84] MCP tool descriptions and server instructions capped at 2KB** —
  Impact: shell-server, state-server, and sliver-server all have `instructions`
  strings passed to FastMCP. If any exceed 2KB, they'll be silently truncated.
  The state-server instructions are likely the longest.
  Action: Audit all MCP server instruction strings for length. Trim if >2KB.

- **[v2.1.84] `TaskCreated` hook added** —
  Impact: red-run could use this to log task creation events, track teammate
  task assignments, or trigger external notifications when the orchestrator
  creates tasks.
  Action: No immediate action needed, but consider adding a TaskCreated hook
  for engagement logging.

- **[v2.1.85] Conditional `if` field for hooks** —
  Impact: red-run's hooks (SubagentStop, TeammateIdle, SessionStart) can now
  be filtered with permission rule syntax. This means the SubagentStop hook
  regex matcher could be simplified, and hooks could be scoped to specific
  tool patterns instead of broad matchers.
  Action: Review hook configurations for potential simplification.

### Medium Impact (monitor)

- **[v2.1.86] `--bare` mode fixed: MCP tools no longer dropped, mid-turn messages no longer discarded** —
  Impact: If red-run ever uses `--bare` mode for teammates or automation,
  this was a critical bug. MCP tools being dropped would make all MCP servers
  invisible. Mid-turn message discarding would break teammate messaging.
  Monitor: Confirm red-run doesn't use `--bare` mode currently.

- **[v2.1.86] Read tool compact line-number format + dedup unchanged re-reads** —
  Impact: Token savings for teammates that read the same files repeatedly
  (common in skill loading, config reading). Net positive for token budget.
  Monitor: No action needed, automatic benefit.

- **[v2.1.86] Skill descriptions capped at 250 chars in `/skills` listing** —
  Impact: red-run's orchestrator skill description is longer than 250 chars.
  Won't affect loading (only the listing view), but verify the truncation
  doesn't hide critical trigger information.
  Monitor: Check `/skills` output for red-run entries.

- **[v2.1.86] Reduced token overhead for `@` file mentions** —
  Impact: Teammates that reference files via `@` mentions use fewer tokens.
  Minor but positive.

- **[v2.1.84] `ANTHROPIC_DEFAULT_*_MODEL_SUPPORTS` env vars** —
  Impact: red-run previously used `ANTHROPIC_DEFAULT_SONNET_MODEL` to control
  teammate model. These new `_SUPPORTS` vars allow overriding capability
  detection (effort levels, thinking). Useful if teammates need different
  thinking modes.
  Monitor: Not needed now, but good to know for future tuning.

- **[v2.1.84] Background bash tasks surface notification after ~45s** —
  Impact: Teammates running long background commands (manspider, enum4linux-ng)
  may see new notifications about "stuck" processes. This is informational
  and shouldn't change behavior, but teammates might react to the notification
  unnecessarily.
  Monitor: Watch for teammates aborting valid long-running commands.

- **[v2.1.84] Rules and skills `paths:` accepts YAML list of globs** —
  Impact: red-run skills could use `paths:` to scope to specific directories.
  Not currently used but available for future skill organization.

- **[v2.1.84] MCP servers deduplicated (local wins over claude.ai)** —
  Impact: If operators have claude.ai MCP connectors that overlap with
  red-run's local MCP servers, local config now takes priority. Correct
  behavior for red-run.

### Low Impact (informational)

- **[v2.1.86] Write/Edit/Read fixed for files outside project root** —
  Relevance: Teammates writing to `~/.claude/` or `/tmp/` should work
  reliably now. Was potentially causing issues with skill loading.

- **[v2.1.86] Config disk write fix (perf + corruption)** —
  Relevance: Skill invocations were causing unnecessary config writes.
  May have contributed to intermittent config corruption.

- **[v2.1.86] Prompt cache improvements for 3P providers** —
  Relevance: Only affects Bedrock/Vertex/Foundry users, not direct API.

- **[v2.1.85] MCP server env vars for headers helpers** —
  Relevance: Useful if red-run ever needs per-server auth headers.

- **[v2.1.85] `/compact` fix for large conversations** —
  Relevance: Long engagements with many teammate messages benefit.

- **[v2.1.84] MCP tool/resource cache leak fix on reconnect** —
  Relevance: SSE MCP servers (shell-server, sliver-server) reconnecting
  after transient failures no longer leak memory in Claude Code.

- **[v2.1.84] Global system-prompt caching works with ToolSearch** —
  Relevance: red-run teammates use ToolSearch for deferred tools. This
  means better prompt caching for all teammates.
